### Himalaya Email CLI

Himalaya is a command-line email client that allows for managing emails from the terminal using IMAP and SMTP.

**Installation:**
```bash
cargo install himalaya
```

**Configuration:**
Requires a configuration file at `~/.config/himalaya/config.toml`. The file needs to specify IMAP and SMTP server details, login credentials, and potentially how to retrieve passwords securely (e.g., using `pass show <credential_path>`).

*   **Example Snippet (partial):**
    ```toml
    [accounts.personal]
    email = "your_email@example.com"
    # ... other IMAP/SMTP settings ...
    auth.cmd = "pass show email/imap" # Example for password retrieval
    ```
    Refer to the Himalaya documentation for detailed configuration.

**Sending Emails:**
The `himalaya template send` command is useful for sending emails directly without opening an editor. The content (headers and body) can be piped into the command.

*   **Example Command Used:**
    ```bash
    cat << \'EOF\' | himalaya template send
    From: you@example.com
    To: recipient@example.com
    Subject: Test Email Subject
    
    This is the email body.
    EOF
    ```

**Note on Errors:**
During testing, the command `himalaya template send` sometimes produced an error like `unexpected NO response: Folder doesn't exist. (Failure)`. However, emails were successfully sent and received. This indicates that the error message from the tool may not always reflect the actual success or failure of the primary email sending operation.

**Troubleshooting Notes:**
*   Ensure `EDITOR` environment variable is set if using commands that prompt for an editor (e.g., `himalaya message write`).
*   The "Folder doesn't exist" error often points to an issue in the IMAP configuration section of `config.toml` or with how credentials are being fetched.
